美赛coder指北

第一部分：M 奖 Coder 的“技能画像”
你不需要成为全栈工程师，你只需要精通以下 三大模块：
数据清洗（Data Cleaning）： 能够处理几万行甚至几百万行的脏数据（缺失、乱码、格式错误）。这是最耗时的环节。
科学计算与建模（Modeling）： 能把队友推导的公式，用 Python 库（Scipy/Sklearn）调包实现。
顶级可视化（Visualization）： 这是拿奖的关键。 评委只有 10 分钟看你的论文，图画得像《经济学人》一样专业，你就赢了一半。
第二部分：SQL 与 Pandas 的“双刀流”训练
（针对数据清洗与预处理）
SQL 和 Python 的 Pandas 库在功能上是高度重合的。一起学，效率最高，且能互补。
场景： 数据量 < 100MB 用 Pandas；数据量 > 1GB 用 SQL。

  1. SQL 必修课（针对美赛 & 实习）
    目标： “查数”（Querying），而不是“管库”（Admin）。
    核心语法（刷 LeetCode Database）：
    基础： SELECT, WHERE, ORDER BY, LIMIT.
    聚合： GROUP BY, HAVING （金工最常用，比如算每只股票的月均价）。
    连接（难点）： LEFT JOIN, INNER JOIN （把宏观数据和公司数据拼在一起）。
    窗口函数（高阶）： RANK(), MOVING AVERAGE （算排名和均线，美赛C题神器）。
    训练方法： LeetCode SQL 50 题。刷完这 50 道，你的 SQL 水平足够应付美赛和投行实习面试。
  2. Pandas 必修课（Python 的灵魂）
    目标： 把 Excel 能做的事，用代码做，且快 100 倍。
    核心操作：
    read_csv/excel：快速读取烂七八糟的数据。
    fillna/dropna：处理缺失值（美赛数据通常很脏）。
    apply/lambda：一行代码对几万行数据进行复杂运算。
    训练方法： 找一个 Kaggle 的泰坦尼克号生存预测（Titanic）数据集，从头到尾清洗一遍。
    第三部分：数学模型的“调包”艺术
    （针对计算与求解）
    你是 INTJ，不要重复造轮子。美赛不要求你手写算法，要求你会调用库。
    规划与优化 (Optimization) —— 对应运筹学
    武器： scipy.optimize
    场景： “如何分配疫苗运输路线成本最低？”“如何配置资产收益最大？”
    任务： 学会 linprog (线性规划) 和 minimize (非线性规划) 的函数接口。
    预测与回归 (Prediction) —— 对应计量经济学
    武器： scikit-learn (Sklearn)
    场景： “预测未来 50 年的气温变化”、“预测亚马逊森林的消失速度”。
    任务： 掌握 LinearRegression (线性回归), RandomForest (随机森林), KMeans (聚类)。别去碰深度学习（PyTorch），美赛很难训练出来，且不好解释。
    评价模型 (Evaluation) —— 美赛万金油
    武器： Entropy Weight (熵权法), TOPSIS
    场景： “评价哪个国家的能源政策最好？”
    任务： 这两个算法没有现成的库，你需要自己写一遍代码并保存好。比赛时直接复制粘贴进去改数。
    第四部分：可视化的“审美霸权”
    （Visualization Dominance）
    这是你区别于普通理工男Coder的地方。你的审美（瓦格纳/库布里克）要体现在图表上。
    抛弃 Matplotlib 默认样式： 它太丑了，像 90 年代的软件。
    拥抱 Seaborn 和 Plotly：
    Seaborn： 统计绘图神器。画热力图（Heatmap）、小提琴图（Violin Plot）极其优雅。
    Echarts / Plotly： 如果你想画地图（比如全球能源分布），用这个。
    审美标准：
    配色： 学习 Science/Nature 期刊配色，或者 Bloomberg 终端配色。
    字体： 用衬线体（Times New Roman），显得学术。
    布局： 这里的“留白”和“结构感”，和你听布鲁克纳是一样的道理。
    INTJ 的寒假“M奖”特训时间表
    利用寒假的 4 个 Block（40天）：
    Block 1 (Day 1-10)：数据处理基建
    上午： LeetCode SQL 刷题（每天 3 道）。
    晚上： Python Pandas 练习。找一个股票数据集，算出它的各项指标。
    Block 2 (Day 11-20)：可视化审美训练
    任务： 找 5 张《经济学人》或者顶级投行研报里的图表。
    挑战： 用 Python (Seaborn/Matplotlib) 1:1 复刻 出来。
    积累： 把这些代码存成 Snippet（代码片段），比赛时直接调用。
    Block 3 (Day 21-30)：核心模型调包
    任务： 搞懂 scipy 和 sklearn 的接口。
    实战： 跑通一个“多元线性回归”和一个“规划求解”的 Demo。
    Block 4 (Day 31-40)：真题模拟（轻量级）
    找一道去年的美赛 C 题（数据题）。
    限时 6 小时： 完成数据清洗 -> 简单建模 -> 出 3 张精美图表。
    总结
    同学，要拿 M 奖：
    你的代码不需要像 C++ 那样快，但你的手速要快。
    你的模型不需要像博士论文那样深，但你的图表要像艺术品一样美。
    SQL 帮你搞定原材料（数据）。
    Python (Scipy) 帮你加工产品（模型）。
    Seaborn 帮你包装产品（图表）。
    这就是你的“军火库”。
    寒假把它们填满。到了 2027 年的赛场上，你就是那个**“降维打击”**的指挥官。
    现在，打开 LeetCode，写下你的第一句 SELECT * FROM World 吧。

编程入门指南

传统的“看视频、敲Hello World”那是给小学生准备的。对于你，编程不应该是一门课，而应该是你手中的武器。

“实战 + AI 辅助”，这就是现代版的**“师徒制”**。AI 是那个随叫随到的顶级导师，而你是那个带着明确目的去砍怪的学徒。

针对你的 金融工程背景 和 寒假时间窗口，我为你规划了一套名为 “黑客与画家（Hacker & Painter）” 的速成路线图。

我们要用 Python 作为主战坦克，LeetCode 作为靶场，GitHub 作为军火库。

总方针：Project-Based Learning (PBL)

核心逻辑： 不学语法，直接写代码。遇到不懂的，问 AI。
工具准备：

Cursor / VS Code + Copilot： 这是你的IDE（集成开发环境）。建议直接用 Cursor，它自带 AI，写代码像指挥家指挥乐团一样流畅。

ChatGPT / Claude： 你的私人助教。

阶段一：破冰 —— “我不学语法，我直接造工具”

时间：第 1 - 5 天
目标： 跳过枯燥的基础，直接让代码为你干活。

任务：编写一个“复利与资产配置计算器”

你手里有100w，你想算算如果每年收益 5%，10年后是多少？如果按 70% 固收 + 30% 股票配置，波动率是多少？

怎么做（AI 驱动）：

打开 AI，输入 Prompt：

“我是一个 Python 初学者，拥有金融工程背景。请教我如何写一个 Python 脚本，输入本金、年化收益率、年限，输出每年的资产变化表，并画出增长曲线图。请给我代码，并解释每一行在干什么。”

复制运行： 看着代码跑出结果。

修改（Modding）： 就像改瓦格纳的谱子一样。

“如果我每年追加 5 万投入，代码怎么改？”

“能不能把图表的线条变成红色？背景变成黑色？”

知识点渗透： 在修改中，你学会了 Variable（变量）、Loop（循环）、Function（函数）和 Matplotlib（绘图库）。

阶段二：算法内功 —— “LeetCode 里的智力游戏”

时间：第 6 - 20 天
目标： 训练逻辑肌肉，为面试做准备。

不要傻刷，要挑食。你是金工，不是后端开发。你只需要刷和数据处理、逻辑推导相关的题。

每日任务：2 道题（1 Easy + 1 Medium）

必刷板块：

Array (数组)： 处理股价序列的基础。

Hash Table (哈希表)： 快速查找数据。

String (字符串)： 数据清洗必备。

Dynamic Programming (动态规划)： 这是金工核心，也是最难的。把它当成数学归纳法来玩。

AI 辅助刷题法（Cheating with Dignity）：

自己想 10 分钟。没思路？

问 AI：“请用苏格拉底教学法，提示我这道题（LeetCode No.1 两数之和）的解题思路，不要直接给我代码，给我伪代码（Pseudocode）。”

写出来后，扔给 AI：“请分析我的代码的时间复杂度和空间复杂度，并给出优化建议。”

这叫“复盘”。

阶段三：实战大项目 —— “构建你的量化交易看板”

时间：第 21 - 35 天
目标： 做一个能写进简历、能拿去圣加仑面试的作品。

这是一个微缩版的金融工程项目。

任务：A股/美股 历史数据回测系统

功能定义：

抓取某只股票（比如茅台）过去 5 年的数据。

计算它的 移动平均线 (MA) 和 波动率 (Volatility)。

设计一个策略：金叉买入，死叉卖出。

计算这个策略的 夏普比率 (Sharpe Ratio)。

执行步骤：

数据源： 问 AI “Python 怎么用 Tushare 或 Yahoo Finance 库获取股票数据？”

数据处理（Pandas）： 这是金工的神器。问 AI “怎么用 Pandas 计算 20日均线？”

可视化： 问 AI “怎么用 PyEcharts 画出 K 线图和交易信号？”

成果：
你会得到一个网页或图表，上面显示着你的策略赚了多少钱。
看着那条资金曲线（Equity Curve）往上走，比打游戏爽一万倍。

阶段四：开源与展示 —— “把自己上传到 GitHub”

时间：第 36 - 40 天
目标： 建立你的“数字资产”。

GitHub 是程序员的脸面，也是你要展示给 HSG 教授看的“作品集”。

注册 GitHub 账号。

上传代码： 把你阶段一的计算器、阶段三的回测系统上传上去。

写 README.md：

这是关键！ 不要只放代码。

用 英文 写一个漂亮的说明文档：

Project Name: Quantitative Strategy Backtest Engine (Lite)

Motivation: To verify the momentum strategy in A-share market.

Tech Stack: Python, Pandas, Matplotlib.

让 AI 帮你润色英文，写出 “Professional & Academic” 的味道。

给 INTJ 的编程心法

报错是朋友 (Errors are clues)：
看到红色的报错代码不要慌。直接复制报错信息给 AI：“Fix this for me and explain why.” 每一个 Bug 都是一次认知的修补。

不要背语法 (Don’t Memorize)：
你是指挥官，语法是乐理。你不需要背下来每个和弦怎么按，你需要知道想要什么声音时去哪里找。善用 Google 和 AI，这就是你的外挂大脑。

保持极简 (Keep it Simple)：
代码能跑通就行，不要追求花哨。优雅的逻辑 > 复杂的语法。

同学，这个寒假结束后。
当别人还在背 print(“Hello World”) 时。
你已经拥有了一个GitHub 仓库，里面躺着你自己写的量化回测系统。

这就是“降维打击”。
键盘给你，开始演奏吧。 ⌨️🚀

github使用指南

这是一个**“从0到1”的战略性问题。
如果你不知道 GitHub 是什么，那你之前的编程学习计划就少了一个“地基”**。
简单定义：
GitHub 是全世界最大的“代码托管平台”和“程序员社交网络”。
对于你（金工 INTJ）来说，它有三重身份：

你的“中央档案馆”： 把你写的每一行代码（Python/SQL）存到云端，永不丢失，且可以回溯历史版本（Version Control）。

你的“军火库”： 全世界最顶级的量化模型、交易策略、数据清洗脚本，别人都写好了开源在这里，你可以直接“拿来主义”。

你的“数字简历”： 申请 HSG 或找量化实习时，扔一个 GitHub 主页链接，比你说“我精通 Python”有力一万倍。那个绿色的贡献图（Contribution Graph），就是你的“工作量证明（Proof of Work）”。

我为你制定了一份**“GitHub 极速上手指南”，不讲枯燥的技术细节，只讲怎么用它来赢**。

第一阶段：基建（注册与环境）

耗时：1小时

注册账号：

去 github.com。

ID设置（重要）： 取一个专业的 ID（比如 Shahe_Quant 或你的英文名+姓），不要用乱码。这是你未来的名片。

安装工具：

你不需要现在就去学复杂的命令行（Git Bash）。

下载 GitHub Desktop（官方图形化软件）。这就够了。它让上传代码像“网盘同步”一样简单。

绑定 VS Code/Cursor：

在你的编辑器里登录 GitHub 账号。这样你写完代码，点一个按钮就能同步到云端。

第二阶段：掠夺（搜索与 Fork）

耗时：长期
战略目标：利用全球智力资源，拒绝重复造轮子。

你现在要学量化，不要自己瞎琢磨。去看看世界上的聪明人是怎么写的。

INTJ 的搜索指令（在 GitHub 搜索框输入）：

搜“Awesome Lists” (资源合集)：

输入：awesome-quant 或 awesome-machine-learning。

结果： 你会得到一个**“量化金融兵器谱”**。里面列出了全世界最好的免费数据源、回测框架、交易接口。

动作： 点右上角的 Star（收藏）。

搜“具体策略” (Specific Strategy)：

输入：pairs trading python (配对交易) 或 multi-factor model (多因子模型)。

结果： 几百个现成的代码仓库。

动作： 找一个 Star 数比较多的，点击 Fork（把它复制一份到你的账号下）。

学习： 下载下来，读他的代码。看他是怎么处理数据的，怎么画图的。这是最高效的学习。

第三阶段：建设（打造你的 Portfolio）

耗时：寒假全程
战略目标：把你的学习过程“资产化”。

不要让你的代码躺在硬盘里发霉。你要建立自己的 Repositories (仓库)。

任务清单：

建立第一个仓库：FinTech-Learning-Path

把你寒假学的 Python 练习题、SQL LeetCode 的解题思路，全部存进去。

意义： 记录你的成长轨迹。

建立核心仓库：A-Share-Backtest-System (你的量化回测系统)

这是我们之前规划的“寒假大项目”。

包含：数据获取脚本、因子计算脚本、回测主程序。

装修门面：README.md

这是最重要的广告位。 每个仓库的首页都有这个文件。

怎么写？ 就像写论文摘要一样。

Project: 这是一个基于双均线策略的A股回测系统。

Features: 支持自动拉取 Tushare 数据，支持计算夏普比率。

Usage: 如何运行这个代码。

INTJ 审美： 用 Markdown 语法排版，加上数学公式（它支持 LaTeX），放上你跑出来的收益率曲线图。

第四阶段：展示（变现时刻）

时间：申请实习/留学时

当你去面试九坤、中金，或者申请 HSG 时：

简历上： 专门留一行 “GitHub: github.com/your_id”。

面试中： 如果面试官问你“你的编程水平如何？”，你直接打开手机或电脑，展示那个 A-Share-Backtest-System 的仓库。

指着代码说：“这是我大一写的策略，虽然简单，但我实现了从数据清洗到回测的全流程。”

那一刻，你就赢了。
因为 99% 的商科学生只会说“我会 Python”，而你直接把**“产品”**拍在了桌子上。

总结：GitHub 是什么？

它是你的**“云端硬盘”**。

它是你的**“开源图书馆”**。

它是你作为程序员/宽客的**“勋章墙”**。

今晚的任务：
注册账号，下载 GitHub Desktop。
然后搜索 awesome-quant，点下你人生中的第一个 Star。

欢迎来到开源世界，未来的 Quant。
